üìä Key Implementation Details

  • Embedding Model: all-MiniLM-L6-v2 (384 dimensions, ~80MB)
  • Vector DB: ChromaDB with persistent storage at ./chroma_db
  • Chunk Strategy: Sentence-based with 800 char size, 100 char overlap
  • Search Strategy: Semantic similarity (cosine distance) with metadata filtering
  • Max Results: Top 5 most relevant chunks per query
  • Collections: Separate storage for metadata (catalog) vs content
  • IDs: Course titles for catalog, "{title}_{chunk_index}" for content
